Aturan Penamaan File dan Code Convention di Go

Aturan Penamaan File

Berikut adalah panduan aturan penamaan file di Go:

1. Gunakan Huruf Kecil Saja

✅ Benar:
main.go
utils.go
database_handler.go

❌ Salah:
Main.go
Utils.go
DatabaseHandler.go

2. Gunakan Underscore (Snake Case) Jika Perlu

✅ Benar:
user_service.go
database_connection.go
http_handler.go

❌ Salah:
userService.go
DatabaseConnection.go
HttpHandler.go

3. Nama File Mewakili Isi atau Tujuan

Contoh:
user_service.go        // Layanan terkait user
database_connection.go // Koneksi database
utils.go               // Fungsi utilitas

4. Gunakan Suffix untuk File Khusus

Contoh:
user_service.go        // Layanan terkait user
user_service_test.go   // Unit test untuk user_service
config_loader.go       // Untuk memuat konfigurasi
http_handler.go        // Handler HTTP

5. File Unit Test Selalu Diakhiri _test.go

✅ Benar:
user_service_test.go
database_handler_test.go

❌ Salah:
user_service_Test.go
test_user_service.go

6. Hindari Nama File yang Duplikat dalam Satu Package

Contoh:
user_service.go
user_repository.go
user_handler.go

7. Nama File Harus Singkat tapi Jelas

✅ Benar:
auth_service.go
db_connector.go
config_loader.go

❌ Salah:
authentication_service_handler.go
database_connection_connector.go
load_the_configuration_file.go

8. Hindari Nama File yang Berkonflik dengan Package Bawaan

❌ Salah:
os.go       // Berkonflik dengan package "os"
strings.go  // Berkonflik dengan package "strings"

✅ Benar:
file_operations.go
string_utils.go

9. Hindari Menggunakan Spasi dan Karakter Khusus

✅ Benar:
user_service.go
user_data.go

❌ Salah:
user service.go
user-data.go
user#service.go

Code Convention di Go Sesuai Standar Google

1. Format Kode dengan gofmt

  • Gunakan gofmt untuk memastikan format kode yang konsisten.
  • Jangan modifikasi secara manual format kode.

2. Nama Variabel dan Fungsi

  • Gunakan camelCase untuk variabel, parameter, dan nama fungsi.
  • Gunakan PascalCase untuk nama fungsi yang diekspos (public).
func calculateTotal() int {   // Private function
    totalAmount := 0          // Variabel dengan camelCase
    return totalAmount
}

func CalculateTax() int {     // Public function
    return 10
}

3. Nama Paket

  • Gunakan huruf kecil tanpa underscore.
  • Nama paket harus deskriptif tetapi pendek.
✅ Benar:
package user
package database

❌ Salah:
package UserData
package database_handler

4. Hindari Variabel yang Tidak Digunakan

  • Go tidak mengizinkan variabel yang dideklarasikan tetapi tidak digunakan.
  • Gunakan underscore (_) jika perlu mengabaikan nilai.
func Example() {
    value, _ := getValue() // Abaikan nilai kedua
}

5. Struktur Kode yang Rapi

  • Kelompokkan kode berdasarkan fungsionalitas.
  • Struktur proyek sebaiknya konsisten.

6. Komentar

  • Gunakan komentar untuk mendokumentasikan fungsi yang diekspos.
  • Komentar harus dimulai dengan nama fungsi/variabel.
// CalculateTax menghitung pajak berdasarkan jumlah total.
func CalculateTax(total int) int {
    return total * 10 / 100
}

Ringkasan

Dengan mengikuti panduan ini, Anda dapat memastikan kode Go Anda lebih konsisten, terorganisir, dan sesuai dengan standar industri seperti yang digunakan oleh Google.
